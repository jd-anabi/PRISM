PROJECT & GOAL
  Simulation-based inference (amortized NPE via the `sbi` package, neural spline flow)
  to recover 16 parameters of the Nadrowski hair-bundle model from a single observable
  x(t): 10 nondimensional (ND) model params + 6 rescale params. Goal: a well-CALIBRATED
  and (where possible) precise posterior, assessed by SBC / TARP / PPC / eye-test.

  Parameter meanings (all defined from dimensional biophysical quantities):

    rescale (6): t_scale=lambda_hb/k_gs (relaxation time), t_offset, x_scale=D (gating
                 swing), x_offset, f_scale, f_offset.
  NOTE: the Reduction / NWK->Hopf branch is being reworked and is IRRELEVANT to SBI.
  Do not invoke or reason about the reduction map for this work.

PIPELINE DESIGN (orienting notes)
  - LATENT reparameterization: the flow trains in an unbounded latent z; T: z->theta maps to the
    physical box (per-param scaled sigmoid). TransformedPosterior wraps the trained latent
    DirectPosterior so .sample/.log_prob speak physical coords. The ND prior is stored as
    TransformedDistribution(latent_GMM, T_nd); build_posterior extracts the latent GMM to train on.
    A trained posterior carries its exact training prior at posterior.prior.gen_dist (a ProductPrior)
    -> the SBC/reparam tools extract it; no separate prior file is needed.
  - TWO-trajectory stats: per sample, simulate a spontaneous (zero-drive) run -> Groups A-F and a
    forced run -> Group G (lock-in to the known drive). A1 (absolute mean) is the only non-demeaned
    feature; everything else is shape/scale-invariant-ish.
  - Conditioning layout = [ S(41) | log(T_obs) | forcing ]; log(T) rides the summary pathway, forcing
    is a separate EmbeddedNet pathway. Keep this order in sync across generate_observations /
    gen_training_data / validate / infer_from_experiment.
  - Scale-batched training: each batch shares a Sobol-sampled (t_scale, T); simulate at fine ND dt
    then downsample to dt_exp; pre-filter by N_ND_MAX.
  - GOTCHA: the latent prior is MIXED-DEVICE (cpu rescale bijection + cuda nd GMM). The pipeline only
    ever SAMPLES it, never .log_prob -- keep it that way (the reparam path also only samples it).

KEY FINDING (the diagnosis driving the current fix)
  The SBC failures are NOT an information / degeneracy deficit -- they are FLOW CALIBRATION.
  Evidence chain:
    - CONVERGED, not under-fit: a convergence retrain plateaued (val loss flat through the full
      early-stop window; small train-val gap) -> more training/capacity won't help.
      (scripts/retrain_convergence.py; the script's auto "under-fit" line is a noise-spike artifact.)
    - DEGENERACY is real but not fatal: dominant aliases kappa~x_scale |cos|=0.94 (amplitude: x(t)
      sees the product D*A_nd) and lambda~t_scale |cos|=0.96 (timescale), in a broadly sloppy
      manifold. (scripts/degeneracy_map.py)
    - BUT identifiability is SUFFICIENT prior-wide: a Laplace marginal-SD analysis (Fisher F=J^TJ from
      sims) shows every param pinned to ~0.3-7% of its prior range, holding across GT + prior samples
      (kappa/lambda/x_scale/t_scale identified in 100% of K=6 points), EXCEPT t_offset & f_scale which
      are genuinely uninformative (SD~1). (scripts/feature_candidate_test.py, identifiability_offgt.py)
  => The params are well-identified, but their joint posterior is a thin, 0.95-correlated ridge that
     the flow mis-calibrates (tight posteriors are hypersensitive to small flow biases in SBC).
     t_offset's mild failure = the flow not perfectly reverting to the prior on an uninformative dim;
     f_scale passes because the flow reverts to prior well there.

THE FIX (Track A) -- IMPLEMENTED + VALIDATED; the retrain is IN PROGRESS (user)

  Fisher eigenbasis V so the correlated posterior becomes axis-aligned and the flow can calibrate it.
    - V = decorrelate.build_latent_fisher_rotation(cfg): from the latent feature-Fisher (sims only,

      model/stats change. Validated by reparam_selftest.py + reparam_wiring_smoke.py (decorrelation
      Fisher off-diag 0.77 -> ~0; bijection round-trip, in-box, log-det preserved; rotated train +
      save/load/sample all pass) -- no retrain needed to confirm correctness.
